How it works

When trying out Svelte, I went to their website, which had an interactive tutorial to learn how Svelte's single file components work. I found this to be very helpful, so I thought I was ready to start using Svelte. I installed the default app from their website, which looked nothing like what I had just been learning, so I looked for a tutorial online. The tutorial I watched cloned a repository from GitHub, which was setup completely different. The default Svelte app's entry point is app.html, which is in the src folder, the content of the body is injected into a div in the body specified by a Svelte variable called %sveltekit.body%. The app from the tutorial has an index.html in the public folder, and there is no indication that anything is injected in the body, but ignoring that, the App.svelte actually looked like what I was learning from the Svelte website interactive tutorial.
